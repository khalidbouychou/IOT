# Part 3: K3d and Argo CD (GitOps)

## Summary
This part of the **Inception-of-Things** project focuses on Continuous Deployment (CD) using **K3d** and **Argo CD**. The goal is to create a cluster that automatically synchronizes with a public GitHub repository.

## Prerequisites
- **Docker Desktop** (Running)
- **K3d** & **Kubectl** (Installed in `/goinfre/$(whoami)/bin`)
- **A Public GitHub Repository** containing the `deployment.yaml` for the playground application.

---

## 🚀 Setup Instructions

### 1. Initialize the Cluster
Run the provided setup script to create the K3d cluster, namespaces, and install Argo CD.
```bash
sh scripts/install.sh
```
*The script will output the initial admin password for Argo CD at the end.*

### 2. Connect the Cluster to your Mac
Ensure your `kubectl` context is pointing to the new cluster:
```bash
k3d kubeconfig merge mycluster --kubeconfig-switch-context
```

### 3. Deploy the Application
Apply the Argo CD Application manifest to link your GitHub repository to the `dev` namespace:
```bash
kubectl apply -f confs/application.yaml
```

## 🛠 Verification & Testing

### 1. Access the Argo CD UI
Since we are on a school network, we use **Port-Forwarding** to reach the UI securely:
```bash
# Run this in a separate terminal
kubectl port-forward -n argocd svc/argocd-server 9090:443
```
- **URL:** `https://localhost:9090` (Accept the security warning)
- **User:** `admin`
- **Password:** *Generated by the install script*

### 2. Test the Application (v1)
Wait for the pod in the `dev` namespace to be `Running`, then test the app:
```bash
# Check status
kubectl get pods -n dev

# Access app
curl http://localhost:8888
```
**Expected Output:** `{"status":"ok", "message": "v1"}`

### 3. The GitOps Demo (v1 ➔ v2)
1. Go to the GitHub repository and edit `deployment.yaml`.
2. Change the image from `wil42/playground:v1` to `wil42/playground:v2`.
3. Commit and push the changes.
4. Observe Argo CD automatically detecting the change and updating the pod.
5. Re-run the test: `curl http://localhost:8888`. It should now return **v2**.
