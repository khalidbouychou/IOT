Goal: Learn GitOps - deploy by git push, not kubectl commands

Setup:
1. K3d cluster runs in Docker (not VirtualBox)
2. Argo CD watches your GitHub repository
3. When you push to GitHub, Argo automatically deploys
4. Uses the 'wil42/playground' Docker image (shows v1 or v2)

Why: Industry standard - deploy by git commit, not manual commands

Initial state: 
  - deployment.yaml has image: wil42/playground:v1
  - Argo CD deploys it → curl localhost:8888 returns "v1"

Change v1 to v2:
  - Edit deployment.yaml in GitHub repo
  - Commit and push
  - Argo CD detects change in ~30 seconds
  - Automatically redeploys
  - curl localhost:8888 now returns "v2"
  
This is GitOps: Git is the source of truth, not kubectl
